# 架构决策记录

本目录保存 Architecture Decision Records，用于记录会影响项目长期边界的技术取舍。

不是每个代码改动都需要 ADR。下面这些情况建议新增或更新 ADR：

- 改变服务的整体代码组织方式。
- 引入会长期影响业务边界的框架、生成器或基础设施依赖。
- 改变数据库、消息、可观测性、配置或部署的核心策略。
- 明确拒绝某种常见方案，而且这个拒绝理由后续容易被遗忘。

文件命名使用 `NNNN-short-title.md`，编号递增。每条 ADR 建议包含：

```markdown
# NNNN. 决策标题

## 状态

已接受 / 已废弃 / 被替代

## 背景

为什么需要做这个决策。

## 决策

最终选择什么方案。

## 后果

这个方案带来的收益、限制和后续影响。

## 替代方案

考虑过但没有选择的方案。
```
